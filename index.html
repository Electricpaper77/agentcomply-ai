Fast path

30-second buyer path

1 Review Policy Pack v1 2 Build a support scenario 3 Preview the audit report 4 Pick the next step

Problem

Customer support agents can create audit risk in one message.

Agents act autonomously

Support agents can look up account data, draft customer emails, and initiate refund workflows with limited human review.

Audit trails are incomplete

Tool logs rarely explain policy context, data sensitivity, approval state, and the reason a decision was made.

Accountability already exists

Teams still need to prove what the agent touched, why the action was allowed, and when escalation or blocking happened.

Buyer objections

Built for the questions buyers ask before a pilot.

Can compliance trust the evidence?

The pilot focuses on normalized synthetic logs, transparent scoring, and repeatable report structure so reviewers can inspect every field.

Will engineering hate the integration?

v0 starts from JSONL-style agent events and a narrow integration point before expanding into broader workflow coverage.

Is this claiming legal compliance?

No. AgentComply AI organizes evidence and risk decisions. It does not provide legal advice, certification, or regulatory approval.

Is this too broad?

The validation path is intentionally narrow: one regulated workflow, one policy library, one schema, and one audit-ready report.

Policy Pack v1

Policy Pack v1: Regulated Customer Support

Four deterministic rules turn synthetic support-agent activity into allow, flag, escalate, or block decisions with inspectable evidence.

PII_OUTBOUND_MESSAGE_REQUIRES_REVIEW

Trigger
Outbound customer message touches PII and lacks human approval.
Required evidence
message_channel, recipient_type, data_touched, human_approved, reviewer_id.
Decision
escalate
Reason code
PII_OUTBOUND_REVIEW_REQUIRED

PAYMENT_OR_REFUND_ABOVE_THRESHOLD_REQUIRES_APPROVAL

Trigger
Refund or payment-adjacent action exceeds the policy threshold.
Required evidence
amount_usd, threshold_usd, approval_status, approver_role, tool_called.
Decision
escalate
Reason code
PAYMENT_THRESHOLD_APPROVAL_REQUIRED

MISSING_AUDIT_FIELDS_BLOCK

Trigger
Required audit fields are absent from the agent event.
Required evidence
event_id, timestamp, agent_id, action_type, tool_called, policy_context.
Decision
block
Reason code
AUDIT_FIELDS_MISSING

PROMPT_INJECTION_OR_UNAUTHORIZED_TOOL_USE_BLOCK

Trigger
Prompt-injection signal or unauthorized tool use appears in the event.
Required evidence
prompt_signal, tool_called, allowed_tools, policy_context, detection_source.
Decision
block
Reason code
UNAUTHORIZED_TOOL_OR_PROMPT_INJECTION
